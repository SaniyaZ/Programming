Я уже когда-то демонстрировал математическую модель дохода сайта. Повторюсь, но приведу более точную модель и опишу параметры модели. Модель создается автоматически - на это уходит 65 секунд даже на мощном компьютере. Точность модели - от 1 до 0.2%, то есть взяв значения параметров за любой период времени (от 30 до 182 дней) в прошлом (до 3 лет в прошлое), модель выдаст результат с указанной точностью.

Вот модель (sp - доход):

 sp = -5.314126e+001
 + 1.816913e+000 * duif*spv/sr
 + 2.549326e-002 * puc*sr*sr*vsr
 - 3.525532e-004 * prt*prt*pu/(spv*spv)
 + 5.878827e-002 * duif*spv/(vsr*vsr)
 + 4.292998e-001 * pug*spv*spv/(prt*prt)
 + 1.369344e-002 * pug*vsr/mpr
 + 2.171506e+001 * duic*mpr*spv*spv
 - 1.299615e+000 * duif*vsr*vsr*vsr*vsr
 - 2.213901e-005 * pug*sr*sr/(mpr*mpr)
 + 2.859366e-005 * pum/(spv*spv*spv*spv)
 + 1.092276e+000 * i/(prt*prt*sr*sr)
 + 5.233347e+001 * duic*mpr*mpr/sr
 - 1.706190e-001 * mpr*puf/spv
 - 4.686151e-005 * duic/(mpr*mpr*prt*prt)
 + 2.829719e-004 * duif*prt/(mpr*mpr)
 + 4.511704e-004 * duif*sr*sr/(prt*prt)
 + 3.505608e-002 * prt*prt*puc*sr
 + 5.130923e-003 * duif*prt*prt*sr*sr
 - 2.426895e-001 * i*prt*prt*prt*prt
 - 3.668434e-003 * duif*vsr*vsr/(spv*spv)

Параметры (все они берутся за одинаковый рассматриваемый период):
duic - количество загрузок коммерческих программ
duif - количество загрузок бесплатных программ
i - количество добавлений программ в корзину (это не продажи)
pu - количество показов всех страниц
puc - количество показов страниц коммерческих программ
puf - количество показов страниц бесплатных программ
pug - количество показов страниц групп программ (эквалайзеры, компрессоры и т.д.)
pum - количество показов главной страницы

mpr - доля показов главной страницы сайта относительно всех показов страниц (например. 0.05)
prt - доля пиратства (например, 0.65)
spv - доход в расчете на одного посетителя сайта (например, 0.30); самый важный параметр, без него невозможно создать настолько точную модель
sr - среднаяя позиция в выдаче поисковых результатов Google (например, 3.5)
vsr - доля "точных" поисковых запросов (например, 0.35); точный запрос означает, что в тексте запроса использовано название сайта или программы на сайте
(параметры mpr, prt, spv, sr и vsr являются корректирующими - они используются для корректировки натуральных показателей)

Зачем это нужно? Модель используется для оптимизации внешнего вида сайта: разным посетителям демонстрируется 2 версии сайта. После сбора статистики применяется модель и проверяется получившийся доход у каждой версии. Та версия сайта, которая приносит наибольший доход, становится основной.
